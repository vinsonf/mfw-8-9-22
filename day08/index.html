GIT flow

Git Flow
reason command
create a new branch git branch branch_name
switch to a branch git checkout branch_name
create a new branch and switch to it git checkout -b branch_name
delete a branch git branch -d branch_name
delete a branch (force) git branch -D branch_name
list all branches git branch
list all branches (remote) git branch -r
list all branches (local and remote) git branch -a
merge a branch into the current branch git merge branch_name
merge a branch into the current branch (abort) git merge --abort
merge a branch into the current branch (continue) git merge --continue
merge a branch into the current branch (no fast-forward) git merge --no-ff branch_name
merge a branch into the current branch (no commit) git merge --no-commit branch_name
merge a branch into the current branch (squash) git merge --squash branch_name
merge a branch into the current branch (strategy) git merge
  1. create project: git init
  2. create feature branch and switch to it: git checkout -b feature-name
  3. create files or update files: new file / touch index.html
  4. add files to staging area: git add .
  5. commit files: git commit -m "message"
  6. push to remote: git push origin feature-name
  7. create pull request on github
  8. merge pull request on github
  9. switch to main branch: git checkout main
  10. pull from remote: git pull origin main
  11. delete feature branch: git branch -d feature-name
  12. Repeat
